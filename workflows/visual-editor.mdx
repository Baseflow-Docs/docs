# Visual Editor

The visual editor lets non-technical team members write, edit, and publish documentation directly in the browser. Every change made in the editor is synced back to your Git repository automatically.

## Inserting Components

Type `/` on a new line to open the component menu. Select a component to insert it:

- **Callout** â€” tip, warning, info, or danger box
- **Card Group** â€” grid of cards
- **Tabs** â€” tabbed content
- **Code Block** â€” syntax-highlighted code
- **Steps** â€” numbered step-by-step guide
- **Accordion** â€” expandable section
- **Image** â€” upload or reference an image

## Page Management

### Creating pages

Click **New Page** in the sidebar to create a new documentation page. Choose where it should appear in the navigation.

### Reordering pages

Drag and drop pages in the sidebar to reorder them. The navigation in `docs.json` is updated automatically.

### Deleting pages

Right-click a page in the sidebar and select **Delete**. The file is removed from the Git repository.

<Callout type="warning">
  Deleting a page in the visual editor permanently removes it from your Git
  repository. This is like deleting a file and committing the change.
</Callout>

## Git Sync

Every action in the visual editor creates a real Git commit:

| Action             | Git Result                                      |
| ------------------ | ----------------------------------------------- |
| Edit page content  | Commit updating the MDX file                    |
| Create new page    | Commit adding a new MDX file + docs.json update |
| Delete page        | Commit removing the file + docs.json update     |
| Reorder navigation | Commit updating docs.json                       |

This means your Git history is always complete â€” even if every edit was made in the browser.

<Callout type="info">
  Developers working in Git and editors working in the browser never conflict.
  Both workflows produce the same MDX files, and Baseflow keeps everything in
  sync.
</Callout>
